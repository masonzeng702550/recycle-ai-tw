# Trashform v2 — Product Requirements Document

> 「以 AI 影像辨識彌補民眾廢棄物分類認知落差——涵蓋日常回收與淨灘場域之在地化指引系統」公民行動提案
> v2 對應提案初稿 How 第 6 項：開發第二版網站。

## 1. 背景與問題

許多民眾有環保意願，卻因缺乏正確的廢棄物辨識/處理知識，且各縣市規則不同、難以在現場即時查詢，導致：

- 日常回收頻繁出現錯誤分類（紙容器混入廢紙、塑膠袋未清洗等）。
- 後端二次分揀人力負擔重，甚至引發工安風險。
- 環境教育素材難以即時更新跨縣市規則，民眾遇到複合材質會放棄查詢。

## 2. 目標使用者

延續提案中的 5 個 Persona，v2 以下列三者為主要驗證對象：

| Persona | 場景 |
|---|---|
| 林鎵（軟體工程師，32）| 外送族，餐後 10 秒內想知道紙餐盒怎麼丟 |
| 尚斑淑（廣告 AE，25）| 忙碌租屋族，需要極簡掃一下就好 |
| 雅琳（環境教育專員，35）| 推薦學員使用，需有跨縣市的權威指引 |

淨灘相關 Persona（斯妤）為 v3 範圍，v2 不處理。

## 3. 範圍與非範圍

### v2 In-Scope
- **跨縣市在地化**：支援臺北市、高雄市兩套處理規則。
- **101 類廢棄物資料庫**：以《回收大百科》收錄之類目為基準。
- **使用者自帶 Gemini API Key**：金鑰僅存於瀏覽器 `localStorage`，不上傳伺服器、不入庫。
- **RWD**：手機（375px 起）、平板、桌機（≥1024px）皆可用。沿用 v1 的 Trashform 視覺識別與深色主題。
- **AI 辨識流程**：拍照/上傳 → 顯示信心度 → 高/中信心顯示處理指引；低信心追問 → 重判；完全無法辨識 → 手動輸入名稱搜尋。
- **錯誤回報**：辨識結果頁可一鍵回報「分錯了」並留下說明（v2 僅蒐集到前端 console / localStorage，不上傳；保留資料結構供 v3 接後端）。

### v2 Out-of-Scope（v3 才做）
- 淨灘模式、海廢/自然物辨識、淨灘軌跡與群組調度。
- 全臺 22 縣市規則。
- 使用者帳號、辨識歷史紀錄持久化、成就機制。
- 自有訓練模型（v2 全程呼叫 Gemini API）。

## 4. 使用者故事

1. 我是林鎵，午餐吃完外送便當，打開網站 → 拍紙餐盒 → **10 秒內**得到「臺北市：一般垃圾（沾油紙容器）」與原因說明。
2. 我是尚斑淑，第一次使用 → 系統提示輸入 Gemini API Key → 我貼上 → 此後不需再輸入即可使用。
3. 我是高雄使用者，在頂端切換縣市為「高雄市」 → 同樣的紙餐盒得到高雄市對應指引。
4. AI 對某張照片只有 50% 信心 → 顯示 2~3 個追問（例：「容器內側是否有透明塑膠膜？」）→ 我選答 → 取得更精準的分類。
5. AI 完全無法辨識 → 我手動輸入「鋁箔包」→ 從 101 類資料庫直接顯示對應指引。
6. 我覺得分錯了 → 點「回報錯誤」→ 留下我認為正確的分類 → 看到「已記錄，謝謝」。

## 5. KPI（沿用提案 6 個月後 KPI，本份對應 v2 階段）

- 完成 v2 MVP 並部署至 Vercel 公開可訪問。
- 累積 ≥ 30 筆實機辨識紀錄（前端 console/log 或自願回填問卷）。
- 跨縣市辨識正確率 ≥ 70%（自有測試集，每縣市 30 張）。
- 至少 5 位使用者完整跑完「拍照→指引→回報」流程並回填回饋表單。

## 6. 主要設計原則

- **零摩擦**：到站第一屏看到「拍照」按鈕，不教學、不註冊。API Key 在第一次點辨識時才彈窗要求。
- **誠實告知信心度**：信心低就說低，不要假裝高信心；追問是常態而非例外。
- **手機優先但桌機不破版**：手機單欄、桌機兩欄（左圖右指引）。
- **離線可瀏覽資料庫**：101 類資料以靜態 JSON bundle 進前端，網路不通也能手動查。
- **隱私**：API Key、圖片、回報內容皆停留在瀏覽器，不過伺服器（Vercel 邊緣節點僅作靜態託管）。

## 7. 風險與待解問題

| 風險 | 緩解 |
|---|---|
| 使用者不知道從哪拿 Gemini API Key | 設定畫面提供 AI Studio 連結與 30 秒文字步驟 |
| API Key 在前端會被洩漏 | 明確告知「金鑰只存你的瀏覽器，請使用個人/免費額度金鑰，勿用機構付費金鑰」 |
| Gemini 免費額度可能限速 | 顯示 429 錯誤並建議稍候，未來可在 v3 加入備援模型 |
| 101 類資料來源版權 | v2 僅引用《回收大百科》分類框架，自行撰寫處理指引文字並標注參考來源 |
| 101 類在 prompt 太長 | 採兩段式：先給 Gemini 大類（10），再用第二次 call 在大類內細分到 101 |
